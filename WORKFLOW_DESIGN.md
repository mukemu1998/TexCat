# TexCat 工作流模式设计

本文档用于规划 `工作流模式 Beta`。目标是在不破坏现有快速工具模式的前提下，把 TexCat 从独立小工具集合逐步升级为游戏贴图生产流水线工具。

## 设计原则

- `main` 分支继续保留稳定快速工具箱，重大改动先在开发分支完成。
- 工作流模式先作为新增入口出现，不替换现有模块。
- 所有步骤都必须可预览、可禁用、可删除、可调整顺序。
- 输出前必须统一做命名预览、目标路径预览、同名冲突检查。
- 默认不修改源文件；只有用户明确选择 `源文件位置` 且通过冲突确认时才写回源图目录。
- 工作流模式复用现有模块能力，优先抽公共逻辑，避免复制一套处理代码。

## 入口结构

TexCat 顶部保留两种模式：

```text
快速工具模式 | 工作流模式 Beta
```

### 快速工具模式

保留当前所有独立模块，适合临时处理单个任务：

- 图像大小
- 格式转换
- 高质量压缩
- 图片裁切
- 法线/黑白图调整
- PBR辅助转换
- 通道拆分
- 通道合并
- 批量重命名

### 工作流模式 Beta

面向完整生产流程：

```text
导入贴图 -> 检查贴图 -> 添加处理步骤 -> 预览输出 -> 执行导出
```

工作流模式初期只支持线性步骤，后续再考虑分支输出。

## 页面布局

推荐三栏结构：

```text
左侧：资源池 / 文件列表 / 批次信息
中间：工作流步骤列表
右侧：步骤参数 / 预览 / 输出摘要
```

### 左侧资源池

显示当前输入文件：

- 文件名
- 贴图类型识别
- 尺寸
- 格式
- 通道模式
- 是否带 Alpha
- 位深提示
- 文件来源：拖入缓存 / 自定义输入目录

### 中间步骤列表

步骤卡片按顺序执行：

```text
步骤 1：裁切
步骤 2：缩放
步骤 3：通道打包
步骤 4：黑白图调整
步骤 5：导出策略
步骤 6：命名规则
```

每个步骤卡片包含：

- 启用/禁用
- 步骤名称
- 摘要参数
- 上移/下移
- 复制步骤
- 删除步骤
- 预览当前步骤输出

### 右侧参数和预览

选中某个步骤后显示对应参数。未选中步骤时显示最终输出摘要。

最终输出摘要包含：

- 预计输出文件数量
- 输出尺寸
- 输出格式
- 输出路径
- 命名结果
- 同名冲突
- Alpha/通道/位深变化提醒

## 工作流步骤类型

### 1. 裁切步骤

复用当前图片裁切模块能力。

参数：

- 单图 / 多图同位置
- 自由框选
- 1:1 锁定
- 自定义像素大小
- 九宫格吸附
- 裁切命名模板

输出：

- 一张或多张裁切后的中间图
- 中间图不写硬盘，除非最后导出

### 2. 缩放步骤

复用图像大小模块。

参数：

- 目标尺寸：256 / 512 / 1K / 2K / 4K / 8K / 自定义
- 保持比例
- 细节优先 / 平衡 / 柔和
- 是否添加尺寸后缀

输出：

- 可生成多个尺寸分支
- 初期为了降低复杂度，多个尺寸可以只在最终导出阶段展开

### 3. 格式与压缩步骤

整合 `格式转换` 和 `高质量压缩`。

参数：

- 输出格式
- 保持原格式
- 无损优先
- 有损格式质量
- JPG/JPEG Alpha 丢失提醒
- DDS 当前能力限制提醒

建议最终成为公共导出策略的一部分，而不是独立步骤长期存在。

### 4. 法线/黑白图调整步骤

复用当前法线/黑白图调整模块。

参数：

- 法线模式：OpenGL / DirectX
- 法线强度
- 黑白强度
- 对比
- 色阶
- Gamma
- 曲线
- 反相

限制：

- 色阶和曲线只作用于黑白图，不作用于法线。

### 5. PBR辅助生成步骤

复用 PBR辅助转换模块。

参数：

- 输入类型：Photo / Normal / Height
- 输出目标：Normal / Height / AO / Cavity / Concavity / Convexity / Curvature 等
- 强度
- 半径
- 细节
- 平滑
- 效果叠加
- OpenGL / DX

初期只允许一个输出目标。后续可支持一次输出多个辅助图。

### 6. 通道拆分步骤

复用通道拆分模块。

参数：

- 输出 L/R/G/B/A 哪些通道
- 每个通道命名模板
- 输出格式

适合从 RGBA 贴图中批量拆出 Mask、Alpha 或打包图通道。

### 7. 通道合并/打包步骤

复用通道合并模块。

参数：

- 基础图
- R/G/B/A 目标通道来源
- 默认黑/白通道
- 来源图指定通道
- 合成图命名

适合 ORM、Mask、Alpha 补通道、RGB/RGBA 通道替换。

### 8. 命名规则步骤

复用批量重命名模块，但应提升为全局能力。

参数：

- 查找替换
- 添加前缀/项目名
- 添加后缀
- 中间插入文本
- `{name}` / `{ext}` / `{type}` / `{channel}` / `{size}` / `{index}` 占位符

建议：

- 工作流中的命名规则应该始终作用在最终输出文件上。
- 独立批量重命名模块继续保留，处理已经存在的文件重命名需求。

## 数据模型草案

工作流可以用 JSON 保存，方便以后做预设和复用。

```json
{
  "version": 1,
  "name": "SP贴图入引擎",
  "input": {
    "mode": "uploaded",
    "source": ""
  },
  "steps": [
    {
      "id": "resize-1",
      "type": "resize",
      "enabled": true,
      "options": {
        "sizes": [2048, 1024],
        "preserve": true,
        "profile": "detail"
      }
    },
    {
      "id": "export-1",
      "type": "export",
      "enabled": true,
      "options": {
        "format": "tga",
        "channel_mode": "auto",
        "quality": 95,
        "lossless": true
      }
    }
  ],
  "naming": {
    "template": "{name}_{size}",
    "steps": []
  },
  "output": {
    "mode": "default",
    "path": ""
  }
}
```

## 推荐内置预设

### 多级尺寸输出

```text
导入 -> 缩放 2K/1K/512/256 -> 命名添加尺寸后缀 -> 导出
```

### SP 贴图入引擎

```text
导入 -> 检查 Alpha/通道 -> 格式转换 TGA/PNG -> 命名规范化 -> 导出
```

### ORM 通道打包

```text
导入 AO/Roughness/Metallic -> 通道合并 R/G/B -> 命名 _ORM -> 导出
```

### 法线批量调整

```text
导入 Normal -> 法线强度/GL-DX -> 格式输出 -> 命名 -> 导出
```

### 黑白遮罩调整

```text
导入 Mask/Roughness/Metallic -> 色阶/Gamma/曲线 -> 灰度 8位 -> 导出
```

### 多图同位置裁切

```text
导入同尺寸多图 -> 裁切同一区域 -> 自动补源图名 -> 导出
```

### PBR辅助图生成

```text
导入 Color/Normal/Height -> 生成 AO/Cavity/Curvature -> 命名 -> 导出
```

## 迭代顺序

### 第一阶段：只做设计和壳

- 增加 `工作流模式 Beta` 入口。
- 增加空的工作流页面布局。
- 不接真实处理逻辑。
- 快速工具模式保持不变。

### 第二阶段：步骤数据结构

- 支持添加、删除、上移、下移步骤。
- 支持保存/载入工作流 JSON。
- 支持最终输出摘要的静态预览。
- 当前 `dev/v1.02` 已完成这一阶段，真实处理逻辑仍暂不接入。

### 第三阶段：接入低风险步骤

优先接入：

- 格式与压缩
- 命名规则
- 缩放

原因：这些步骤比较接近现有批处理逻辑，风险低。

当前 `dev/v1.02` 已开始落地这一阶段：工作流右侧参数面板先支持缩放尺寸、格式与压缩、命名规则，可读取当前快捷模块设置并保存到工作流 JSON。完整流水线执行仍留到后续阶段。

### 第四阶段：接入交互型步骤

接入：

- 裁切
- 通道拆分
- 通道合并

这些步骤需要更多预览状态和中间数据，风险较高。

### 第五阶段：接入生成型步骤

接入：

- 法线/黑白图调整
- PBR辅助生成

这些步骤处理成本更高，预览和性能需要单独优化。

## 风险和回滚

- `v1.01` 标签是当前稳定回滚点。
- 工作流模式先在 `dev/v1.02` 或后续开发分支推进。
- 每完成一个阶段都提交一次，避免大范围不可控变更。
- 工作流模式出现问题时，不影响快速工具模式。
- 任何公共导出策略重构前，必须先保留当前处理路径或增加兼容层。

## 暂不做的内容

- 不做云端处理。
- 不做联网素材库。
- 不做专有 BC1/BC3/BC7 DDS 编码承诺。
- 不在 Beta 阶段移除现有独立模块。
- 不在初期支持复杂分支图形节点编辑器。

工作流模式的第一目标是让游戏贴图生产流程更顺，不是把 TexCat 变成复杂节点软件。
